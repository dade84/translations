# 0.3 The Devil and the Details: Syntax #

## Знакомство с разделителем do-end ##

Блок может быть создан обернув часть кода в фигурный скобки ( {} ) или слова do и end. В отличие от фигурных скобок, которые вы видели до сих пор, do - end синтаксис требует чтобы do, код блока и end все были на отдельных строчках.

Как всегда, позвольте мне продемострировать с помощью четырех примеров, каждый из которых делает тоже самое, но используя различный синтаксис. Не сомневайтесь экспериментировать с ними - для того они тут.

	addition = lambda {|a, b| a + b }
	puts addition.call(5, 5)

	addition = lambda {|a, b|
	  a + b
	}
	puts addition.call(5, 5)

	addition = lambda do |a, b|
	  a + b
	end
	puts addition.call(5, 5)

	addition = lambda do |a, b|; a + b; end
	puts addition.call(5, 5)

Поскольку все эти формы функционально одинаковы, мы опускаемся до социальных норм, когда приходится определяться между ними. В большинестве репозиториях Ruby кода, соглашение использовать фигурные скобки для блоков только для однострочного кода и do-end для многострочного.

Пожалуйста, имейте ввиду, что последний пример только для демонстрации того, что do-end форма все еще работает, когда вы создаете разрывы строк "искусствено" используя соответствующий Ruby синтаксис (точка запятая). Тем не менее, эта форма явно громоздкой чтобы писать и я сомневаюсь что вы на самом деле видели такое использование где-либо.
